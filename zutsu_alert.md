# プライバシーポリシー

**頭痛アラート（zutsuAlert）**

最終更新日：2026年5月6日
アプリ名：頭痛アラート（zutsuAlert）

## 1. はじめに

頭痛アラート（以下、「本アプリ」）は、気圧変化から頭痛リスクを予測・可視化し、同じ症状を持つユーザー同士で体験を共有できるモバイルアプリです。本プライバシーポリシーは、本アプリをご利用いただく際に、ユーザーの情報がどのように取り扱われるかについて説明するものです。

**本アプリは健康関連情報（頭痛記録）を扱うため、特に慎重に情報を取り扱います。**

## 2. 取得する情報

本アプリは、以下の情報を取得・利用します：

### 2.1 ユーザーが提供する情報

- **表示名**：アプリ内で使用するニックネーム
- **ユーザーID**：@usernameの形式で他のユーザーを識別するID
- **プロフィール画像**：任意で設定できるアイコン画像
- **自己紹介文**：プロフィールに表示される短い紹介文（任意）
- **頭痛記録**：以下を含む構造化された記録
  - 発生時刻
  - 痛みの強さ（5段階）
  - 服薬の有無、服薬名（任意）
  - 痛む場所（前頭部・こめかみ・後頭部・全体）（任意）
  - 症状（吐き気・めまい・光過敏・音過敏・耳鳴り・倦怠感）（任意）
  - メモ（任意）
  - 公開範囲（公開／フォロワーのみ／自分のみ）
- **フリー投稿**：自由なテキスト投稿（最大280文字）と添付画像（最大4枚）
- **コメント**：投稿へのコメント
- **いいね**：投稿への評価
- **フォロー情報**：他のユーザーとのフォロー関係
- **ブロック情報**：ブロックしたユーザーの記録
- **通報情報**：不適切な投稿の通報内容

### 2.2 位置情報

- **現在地情報**：GPSによる位置情報（緯度・経度）
- **ホーム位置情報**：朝の予報通知用に設定する位置（任意）
- **頭痛発生時の位置情報**：頭痛記録に紐づく地理グリッドID

位置情報は、以下の目的で使用されます：
- 現在地の気圧データを取得して頭痛リスクを表示するため
- 朝の予報通知を該当地域の気圧から生成するため
- 頭痛と気圧の関係性を個別に学習するため

**位置情報は緯度経度を 0.1 度単位（約11km四方）に丸めて利用しており、ピンポイントの居場所を特定する形では取得しません。**

### 2.3 カメラ・写真

- **プロフィール画像**：ユーザーが設定するアイコン画像
- **フリー投稿の添付画像**：投稿に添付する写真

カメラ／フォトライブラリへのアクセス権限は任意であり、拒否しても基本機能（頭痛記録、テキスト投稿など）は利用できます。

### 2.4 自動的に収集される情報

- **デバイス情報**：端末の種類、OSのバージョン、デバイスモデル、アプリのバージョン
- **利用状況データ**：アプリの起動回数、機能の使用頻度、頭痛記録の累積数（個別補正アルゴリズム用）
- **クラッシュレポート**：エラー発生時のログ情報（Firebase Crashlytics）
- **プッシュ通知トークン**：通知送信に必要な識別子（FCMトークン）
- **気圧感受性データ**：頭痛発生時の気圧値・変化幅から自動算出される個人別の閾値

**本アプリは、ユーザーの実名・電話番号・住所などの個人を直接特定できる情報を収集しません。** メールアドレスは、メールでログインを選択した場合のみ Firebase Authentication が管理します。

## 3. 健康関連情報の取り扱い

本アプリは、頭痛記録という健康関連情報を扱います。以下のとおり特に慎重に取り扱います。

- **個人を特定できる形での第三者提供は一切行いません**
- 学術研究目的で匿名統計化されたデータを提供する場合は、事前にユーザーの同意を得ます
- ユーザーは記録ごとに「公開／フォロワーのみ／自分のみ」から公開範囲を選択できます
- 個別補正アルゴリズムは、本人のデータのみを用いて本人の頭痛リスク予測精度を向上させます

## 4. 情報の利用目的

収集した情報は、以下の目的で利用されます：

- 気圧データから頭痛リスクを予測・表示するため
- 個別補正アルゴリズムによってユーザー個人の精度を向上させるため
- 朝の予報通知・気圧急変通知・個別予兆通知を配信するため
- 頭痛記録のフィード表示およびコミュニティ機能を提供するため
- フリー投稿・コメント・いいねによるユーザー間の交流促進
- フォロー機能による関心のあるユーザーの投稿追跡
- アプリの品質向上と不具合の修正
- 利用状況の分析とサービスの改善
- 不正利用の防止とセキュリティの維持
- 不適切な投稿への対応（通報機能）
- ユーザー体験の向上（ブロック機能）

## 5. 情報の保存

### 5.1 データの保存場所

- ユーザーデータ、頭痛記録、フリー投稿は **Google Firebase Firestore** に保存されます
- 認証情報は **Firebase Authentication** で管理されます（匿名認証 / Google / Apple / メール）
- 画像データは **Firebase Cloud Storage** に保存されます
- 利用状況データは **Firebase Analytics** で収集されます
- プッシュ通知は **Firebase Cloud Messaging (FCM)** で送信されます
- 気圧データは **OpenWeather API** から取得し Firebase Firestore にキャッシュされます

### 5.2 データの削除

- アプリ内の「アカウント削除」機能により、以下のデータを削除できます：
  - ユーザーアカウント情報
  - すべての頭痛記録
  - 投稿したフリー投稿
  - 投稿したコメント
  - いいね情報
  - フォロー・フォロワー情報
  - ブロック情報
  - 保存された画像

ただし、以下の情報は、サービスの適切な運営のために一定期間保持される場合があります：
- 不正利用防止のための最小限のログ情報
- 通報情報（運営による確認が完了するまで）
- 法令により保存が義務付けられている情報

## 6. 情報の共有

### 6.1 アプリ内での情報共有

本アプリでは、以下の情報が他のユーザーに公開される可能性があります（公開範囲設定により制御可能）：

- **頭痛記録（公開設定時）**：発生時刻、痛みの強さ、服薬有無、症状、メモ、気圧スナップショット、投稿者情報
- **フリー投稿**：本文、添付画像、投稿日時、投稿者情報、いいね数、コメント数
- **コメント**：投稿に対するコメント
- **プロフィール情報**：表示名、ユーザーID、プロフィール画像、自己紹介、フォロー・フォロワー数
- **フォロー情報**：フォロー・フォロワーのリスト

**ブロック機能**を利用すると、特定のユーザーの投稿を非表示にできます。ブロックされたユーザーには通知されません。

### 6.2 第三者への提供

本アプリは、以下の場合を除き、ユーザーの情報を第三者に提供しません：

- 法令に基づく開示要請があった場合
- 人の生命、身体または財産の保護のために必要があり、本人の同意を得ることが困難な場合
- ユーザーの同意を得た場合

## 7. 第三者サービスの利用

本アプリでは、以下の第三者サービスを利用しています：

- **Google Firebase**（Firestore、Authentication、Cloud Storage、Cloud Functions、Analytics、Cloud Messaging、Crashlytics）
- **OpenWeather**（気圧・天気データ取得）
- **Google Sign-In**（ログイン）
- **Apple Sign In**（ログイン）
- **Google AdMob**（広告配信）
- **Apple App Tracking Transparency**（iOS 広告トラッキング許諾）

これらのサービスのプライバシーポリシーについては、以下をご参照ください：

- Firebase プライバシーポリシー: https://firebase.google.com/support/privacy
- Google プライバシーポリシー: https://policies.google.com/privacy
- OpenWeather プライバシーポリシー: https://openweather.co.uk/privacy-policy
- Apple プライバシーポリシー: https://www.apple.com/legal/privacy/

## 8. データセキュリティ

本アプリは、ユーザーデータを保護するため、以下のセキュリティ対策を実施しています：

- Firebase セキュリティルールによるアクセス制御
- HTTPS通信による暗号化
- 匿名認証によるプライバシー保護（任意でアカウント連携可能）
- ブロック機能・通報機能による不適切な投稿への対策
- OpenWeather API キーは Secret Manager で管理し、クライアントから直接呼び出さない設計

## 9. ユーザーの権利

ユーザーは以下の権利を有します：

- **データへのアクセス**：自分のデータを確認する権利
- **データの修正**：プロフィール情報や投稿内容を修正する権利
  - 表示名・ユーザーIDの変更（重複していない範囲で自由に）
  - プロフィール画像の変更
  - 頭痛記録・フリー投稿の編集・削除
- **データの削除**：アカウントとすべてのデータを削除する権利
- **通知設定**：朝の予報・リスク急変・個別予兆・SNS活動などカテゴリ別に通知ON/OFFを設定する権利
- **公開範囲の設定**：頭痛記録ごとに「公開／フォロワーのみ／自分のみ」を選択する権利
- **ブロック機能**：特定のユーザーの投稿を非表示にする権利
- **通報機能**：不適切な投稿を運営に通報する権利
- **位置情報の制御**：端末設定からいつでも位置情報の権限を変更する権利

## 10. 位置情報の取り扱い

### 10.1 位置情報の利用

- 現在地の気圧データ取得（緯度経度を 0.1 度単位に丸めて使用）
- 朝の予報通知用のホーム位置設定
- 頭痛記録に紐づく地理グリッドID（個別補正用）

### 10.2 位置情報の権限

本アプリは、以下の位置情報権限を使用します：

- **iOS**: 「このAppの使用中のみ許可」または「常に許可」
- **Android**: 「アプリの使用中のみ」または「常に許可」

位置情報の権限は、端末の設定からいつでも変更できます。位置情報を無効にすると、本アプリの主要機能（気圧予測）が利用できなくなります。
